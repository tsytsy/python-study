一、nginx
1. nginx的config的配置学习
可以使用rpm -qa | grep nginx来查看nginx软件都安装在哪些位置

linux可执行程序在/usr/bin，
数据文件和帮助在/usr/share下，
运行时加载的配置文件却又是从/etc下读取，
用户使用它编辑的内容则放在/home下面自己的目录中。

/etc/nginx/nginx.conf
/etc/nginx/nginx.conf.default
这两个文件是nginx的配置文件


/etc/rc.d/init.d/nginx这个文件说明nginx成了一个service，可以使用service nginx start | stop | reload等等命令来进行控制


/usr/sbin/nginx此文件为nginx的可执行文件

/var/log/nginx为nginx日志文件

下面重点说一下配置文件

 server {
        listen       80;
        server_name  localhost;

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
            root   html;
            index  index.html index.htm;
        }



在配置文件中以上部分表示服务器监听80端口，服务器的名称为localhost，nginx的根目录为html，即/usr/share/nginx/html/

当客户端访问服务器的nginx服务时，假设我的电脑做nginx服务器，IP为192.168.1.10，另外一台电脑做客户端，IP为192.168.1.2
那么当客户端打开浏览器，输入http://192.168.1.10:80 后，此时服务器会在nginx的根目录html目录下发送一个默认的index.html
返回给客户端，假设客户端此时在浏览器输入的是http://192.168.1.10:80/a/b/c ，那么此时则会返回一个404 not found字段给客户端
因为服务器没有这个目录，此时如果你在/usr/share/nginx/html/目录下新建一个/a/b/c.txt的文化，那么浏览器再次访问的时候服务器
就会找到这个文件，并且以http协议返回给浏览器，这就是前端到后台的一个简单回路。
当然nginx还可以要买回来做反向代理，用来做负载均衡。
http://nginx.org/en/docs/stream/ngx_stream_upstream_module.html
